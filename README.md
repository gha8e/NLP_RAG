نظام مساعد ذكي متخصص في معالجة اللغة الطبيعية (NLP)
1. مقدمة المشروع
يهدف هذا المشروع إلى بناء نظام ذكي متخصص في مجال معالجة اللغة الطبيعية (Natural Language Processing) باستخدام نماذج مفتوحة المصدر تعمل محليًا، دون الاعتماد على أي خدمات سحابية أو واجهات برمجية مغلقة.
يعتمد النظام على بنية الاسترجاع المعزز بالتوليد (Retrieval-Augmented Generation – RAG)، حيث يتم ربط نموذج لغوي بقاعدة معرفة نصية خاصة، لتمكين النظام من الإجابة على أسئلة المستخدمين ضمن نطاق معرفي محدد بدقة وموثوقية أعلى.
2. أهداف المشروع
فهم أسئلة المستخدم المكتوبة بلغة طبيعية
استرجاع معلومات ذات صلة من مصدر معرفي خاص
توليد إجابات مرتبطة بالمجال المحدد
تقليل ظاهرة الهلوسة من خلال الاعتماد على الاسترجاع
تشغيل النظام بالكامل بشكل محلي باستخدام نماذج مفتوحة المصدر
3. بنية النظام (RAG Architecture)
يتكون النظام من المراحل التالية:
تنظيف ومعالجة النصوص (Text Preprocessing)
تقسيم النص إلى مقاطع (Chunking)
توليد المتجهات الدلالية (Embeddings)
بناء قاعدة بيانات متجهية (Vector Database)
استرجاع المقاطع الأكثر تشابهًا
بناء Prompt مخصص للإجابة
توليد الإجابة النهائية
4. معالجة وتنظيف البيانات
تم تنفيذ مرحلة تنظيف النصوص قبل توليد الـ Embeddings، وتشمل:
إزالة الرموز غير الضرورية
إزالة الفراغات الزائدة
توحيد الصيغ (Lowercasing / Normalization)
Tokenization
حذف الكلمات الشائعة عند الحاجة
مثال:
قبل التنظيف:
نسخ التعليمات البرمجية

NLP!!! is    very   useful!!!
بعد التنظيف:
نسخ التعليمات البرمجية

nlp is very useful
5. البيانات (Dataset)
بيانات نصية خاصة بمجال معالجة اللغة الطبيعية
يتم استخدامها كمصدر معرفي داخلي للنظام
لا يتم إرسال البيانات لأي جهة خارجية
6. نموذج الـ Embeddings
النموذج المستخدم:
sentence-transformers/all-MiniLM-L6-v2
نموذج مفتوح المصدر
يعمل محليًا
أبعاد المتجهات: 384
7. قاعدة البيانات المتجهية
تم استخدام مكتبة FAISS
يتم تخزين المتجهات محليًا
تُستخدم للبحث الدلالي واسترجاع المعرفة
8. النموذج اللغوي
نموذج لغوي مفتوح المصدر
يعمل محليًا دون استخدام أي API خارجي
لم يتم استخدام:
ChatGPT API
Gemini API
Claude API
9. اختبار وتقييم النظام
تم إعداد 20 سؤال اختبار
تشمل:
أسئلة داخل المجال
أسئلة خارج المجال
تم تقييم النظام بناءً على:
دقة الإجابة
ارتباطها بالمجال
تقليل الهلوسة
10. واجهة الاستخدام
تم بناء واجهة بسيطة باستخدام Streamlit
تتيح للمستخدم إدخال الأسئلة والحصول على الإجابات مباشرة
11. طريقة التشغيل

13. الخلاصة
يوضح هذا المشروع القدرة على بناء نظام NLP واقعي يعمل محليًا باستخدام نماذج مفتوحة المصدر، وتطبيق مفهوم RAG بشكل عملي، مع الالتزام الكامل بالمعايير الأكاديمية ومتطلبات المشروع.
